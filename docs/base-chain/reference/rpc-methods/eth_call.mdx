Executes a message call against the current chain state without broadcasting a transaction. No gas is consumed on-chain. Used to read contract state or simulate calls.

## Parameters

<ParamField body="transactionObject" type="object" required>
  The transaction call object.

  <Expandable title="Transaction object fields">
    <ParamField body="from" type="string">
      The 20-byte address the call is sent from. Optional; defaults to the zero address.
    </ParamField>
    <ParamField body="to" type="string">
      The 20-byte address of the contract to call.
    </ParamField>
    <ParamField body="gas" type="string">
      Hex-encoded integer of gas to provide for the call. Defaults to a high limit if omitted.
    </ParamField>
    <ParamField body="gasPrice" type="string">
      Hex-encoded gas price in wei. For legacy transactions. Optional.
    </ParamField>
    <ParamField body="maxFeePerGas" type="string">
      EIP-1559 maximum total fee per gas. Optional.
    </ParamField>
    <ParamField body="maxPriorityFeePerGas" type="string">
      EIP-1559 maximum priority fee per gas. Optional.
    </ParamField>
    <ParamField body="value" type="string">
      Hex-encoded ETH value to send with the call in wei. Optional.
    </ParamField>
    <ParamField body="data" type="string">
      ABI-encoded call data: the 4-byte function selector followed by encoded arguments. Optional.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="blockParameter" type="string" required>
  Block number in hexadecimal, or a block tag: `latest`, `earliest`, `pending`, `safe`, or `finalized`.
</ParamField>

## Returns

<ResponseField name="result" type="string">
  The return value of the executed contract call as a hex-encoded byte array.
</ResponseField>

## Flashblocks Behavior

On a [Flashblocks-aware endpoint](/base-chain/flashblocks/apps#rpc-endpoints), use `"pending"` as the block parameter to query preconfirmed state — data available up to 200ms before the next full block is sealed.

Executes the call against the latest preconfirmed Flashblock state. Use `"pending"` to simulate against the most current chain state, including transactions not yet in a sealed block.

<CodeGroup>
```bash cURL (preconfirmed state)
curl https://mainnet-preconf.base.org \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_call","params":[{"to":"0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913","data":"0x70a08231000000000000000000000000d3CdA913deB6f4967b2Ef66ae97DE114a83bcc01"},"pending"],"id":1}'
```
</CodeGroup>

## Example

<CodeGroup>
```json Request
{
  "jsonrpc": "2.0",
  "method": "eth_call",
  "params": [
    {
      "to": "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
      "data": "0x70a082310000000000000000000000004200000000000000000000000000000000000006"
    },
    "latest"
  ],
  "id": 1
}
```

```json Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": "0x0000000000000000000000000000000000000000000000000000000005f5e100"
}
```
</CodeGroup>

## Errors

| Code | Message | Description |
| --- | --- | --- |
| `-32000` | execution reverted | The call reverted. The `data` field in the error object contains the ABI-encoded revert reason when available. |
