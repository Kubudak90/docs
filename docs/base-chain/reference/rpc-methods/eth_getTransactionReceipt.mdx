Returns the receipt for a mined transaction. Receipts are only available after the transaction has been included in a block.

<Info>
For preconfirmed transaction receipts before a block is sealed, use a [Flashblocks-aware endpoint](/base-chain/flashblocks/apps#rpc-endpoints) and see the [Flashblocks Behavior](#flashblocks-behavior) section below.
</Info>

## Parameters

<ParamField body="transactionHash" type="string" required>
  The 32-byte hash of the transaction.
</ParamField>

## Returns

<ResponseField name="result" type="object | null">
  A transaction receipt object, or `null` if the transaction is not found or is still pending.

  <Expandable title="Transaction receipt fields">
    <ResponseField name="transactionHash" type="string">
      32-byte hash of the transaction.
    </ResponseField>
    <ResponseField name="transactionIndex" type="string">
      Transaction's index position in the block.
    </ResponseField>
    <ResponseField name="blockHash" type="string">
      32-byte hash of the block this transaction is in.
    </ResponseField>
    <ResponseField name="blockNumber" type="string">
      Block number this transaction is in.
    </ResponseField>
    <ResponseField name="from" type="string">
      20-byte address of the sender.
    </ResponseField>
    <ResponseField name="to" type="string">
      20-byte address of the receiver. `null` for contract creation transactions.
    </ResponseField>
    <ResponseField name="cumulativeGasUsed" type="string">
      Total gas used in the block at the point this transaction was executed.
    </ResponseField>
    <ResponseField name="effectiveGasPrice" type="string">
      Actual gas price paid per unit of gas for this transaction.
    </ResponseField>
    <ResponseField name="gasUsed" type="string">
      Gas used by this specific transaction.
    </ResponseField>
    <ResponseField name="contractAddress" type="string | null">
      The address of the contract created by this transaction, if it was a contract creation. Otherwise `null`.
    </ResponseField>
    <ResponseField name="logs" type="array">
      Array of log objects generated by this transaction.
    </ResponseField>
    <ResponseField name="logsBloom" type="string">
      256-byte bloom filter for the logs of this transaction.
    </ResponseField>
    <ResponseField name="status" type="string">
      `"0x1"` for a successful transaction, `"0x0"` for a failed (reverted) transaction.
    </ResponseField>
    <ResponseField name="type" type="string">
      Transaction type: `"0x0"` (legacy), `"0x1"` (EIP-2930), `"0x2"` (EIP-1559).
    </ResponseField>
  </Expandable>
</ResponseField>

## Flashblocks Behavior

On a [Flashblocks-aware endpoint](/base-chain/flashblocks/apps#rpc-endpoints), use `"pending"` as the block parameter to query preconfirmed state — data available up to 200ms before the next full block is sealed.

Returns the receipt for a preconfirmed transaction before it is included in a finalized block. Only available for transactions already included in a Flashblock.

<CodeGroup>
```bash cURL (preconfirmed state)
curl https://mainnet-preconf.base.org \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_getTransactionReceipt","params":["0xb903239f8543d04b5dc1ba6579132b143087c68db1b2168786408fcbce568238"],"id":1}'
```
</CodeGroup>

## Errors

| Code | Message | Description |
| --- | --- | --- |
| `-32000` | transaction indexing is in progress | The node is still indexing transactions. Retry after the node has finished syncing. |
