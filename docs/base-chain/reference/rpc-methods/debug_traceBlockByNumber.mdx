<Info>
Debug methods require a node with debug APIs enabled. Availability and rate limits vary by [node provider](/base-chain/tools/node-providers). These methods replay transactions and are computationally expensive — avoid calling them in hot paths.
</Info>

Replays all transactions in a block identified by its number and returns an execution trace for each transaction.

## Parameters

<ParamField body="blockParameter" type="string" required>
  Block number in hexadecimal, or a block tag: `latest`, `earliest`, `safe`, or `finalized`.
</ParamField>

<ParamField body="traceOptions" type="object">
  Optional trace configuration. Accepts the same fields as [`debug_traceTransaction`](/base-chain/reference/rpc-methods/debug_traceTransaction).
</ParamField>

## Returns

<ResponseField name="result" type="array">
  An array of trace result objects, one per transaction in the block. Same format as [`debug_traceBlockByHash`](/base-chain/reference/rpc-methods/debug_traceBlockByHash).
</ResponseField>

## Example

<CodeGroup>
```json Request
{
  "jsonrpc": "2.0",
  "method": "debug_traceBlockByNumber",
  "params": ["latest", { "tracer": "callTracer" }],
  "id": 1
}
```
